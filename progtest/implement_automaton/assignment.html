Druhý úkol

Úkolem je implementace automatu z první části úkolu

Minimalizovaný automat z první úlohy implementujte v podobě funkce programu v C++, jejíž signatura je následující: 

/**
 * students' interface
 * @param inputString null terminated string with automaton input
 * @return true if string has been accepted
 */
bool checkString(const char * inputString);

Vstupem je řetězec 8bitových znaků libovolné délky, zakončený nulovým znakem. Výstupem funkce je hodnota true nebo false, podle toho, zda je vstup automatem přijat.

Odevzdává se jeden zkompilovatelný zdrojový soubor a ten bude zpracováván stejnými mechanismy jako úlohy na PA1/2. Konfliktní názvy (např. funkce main) je tedy vhodné opatřit podmíněným překladem pomocí makra __PROGTEST__. Např.

#ifndef __PROGTEST__

int main() {
    using namespace std;
    string line;
    while (getline(cin, line)){
        cout << checkString(line.c_str()) << endl;
    }
}
#endif
Svým funkcím a proměnným přidělujte pokud možno unikátní identifikátory, kterým nehrozí konflikt s funkcemi ze standardní knihovny. Můžete je např. začínat prefixem aag_. Pokud budete dodržovat tuto radu, nemůžete se během kompilace na progtestu dostat do jmenných konfliktů.
